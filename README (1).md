# מסלול·התמחות — מדריך פריסה מפורט, צעד־אחר־צעד

**מה יהיה לך בסוף:** כתובת אינטרנט ציבורית כמו
`https://raphael-md.github.io/residency-tracker/`
שכל חבר יכול להיכנס אליה, להתחבר עם חשבון ה-Google שלו, ולראות את ההתקדמות של כולם בזמן אמת.

**מה צריך מראש:** חשבון Google (יש לך), חשבון GitHub (ניצור בדרך אם אין), הקובץ `index.html` שקיבלת, והקובץ `firestore.rules`.

**זמן כולל:** כ-15 דקות. עושים את זה פעם אחת בלבד.

**המבנה הכללי:** האתר עצמו יושב ב-GitHub (חינם). ההתחברות עם Google והנתונים המשותפים יושבים ב-Firebase של Google (חינם). שני השלבים הראשונים מקימים את Firebase, שני האחרונים מעלים את האתר ומחברים ביניהם.

---

## חלק א׳ — הקמת Firebase (הצד של Google)

### שלב 1: יצירת פרויקט

1. פתח בדפדפן: **https://console.firebase.google.com**
2. התחבר עם חשבון ה-Google שלך אם תתבקש.
3. לחץ על הכרטיס הגדול **"Create a project"** (או "Add project" אם כבר יש לך פרויקטים).
4. במסך "Name your project": הקלד `residency-tracker` → לחץ **Continue**.
5. במסך הבא, אם מוצע לך Gemini/AI features — אפשר להשאיר או לכבות, לא משנה → **Continue**.
6. במסך "Google Analytics": כבה את המתג (**Disable/off**) — לא צריך את זה → לחץ **Create project**.
7. המתן כ-30 שניות עד "Your new project is ready" → לחץ **Continue**.

עכשיו אתה בתוך לוח הבקרה של הפרויקט.

### שלב 2: הפעלת התחברות עם Google

1. בתפריט השמאלי, לחץ על **Build** (ייפתחו תת-סעיפים) → לחץ **Authentication**.
2. לחץ על הכפתור הכחול **Get started**.
3. תגיע ללשונית **Sign-in method**. ברשימת הספקים (Additional providers) לחץ על **Google**.
4. במסך שנפתח:
   - הזז את המתג העליון ל-**Enable** (יהפוך כחול).
   - בשדה **Support email for project** — בחר את האימייל שלך מהרשימה.
5. לחץ **Save**.

תראה עכשיו את Google עם סטטוס ירוק "Enabled". ✅

### שלב 3: יצירת מסד הנתונים (Firestore)

1. בתפריט השמאלי: **Build → Firestore Database**.
2. לחץ **Create database**.
3. **Location**: בחר `eur3 (europe-west)` — הכי קרוב לישראל. (אי אפשר לשנות אחר כך, אבל זה לא קריטי.) → **Next**.
4. **Secure rules**: השאר על **Start in production mode** → לחץ **Create**.
5. המתן עד שהמסד נוצר — תראה טבלה ריקה.

### שלב 4: הדבקת חוקי האבטחה

בלי השלב הזה אף אחד לא יוכל לשמור פרופיל.

1. בתוך מסך ה-Firestore, למעלה, לחץ על הלשונית **Rules**.
2. תראה עורך טקסט עם כמה שורות קוד. **סמן הכל ומחק** (Ctrl+A ואז Delete).
3. פתח את הקובץ `firestore.rules` שקיבלת ממני (בפנקס רשימות / Notepad), העתק את כל התוכן שלו, והדבק בעורך.
4. לחץ על הכפתור **Publish** (למעלה).
5. אמורה להופיע הודעה ירוקה שהחוקים פורסמו.

### שלב 5: קבלת "תעודת הזהות" של הפרויקט (firebaseConfig)

1. לחץ על גלגל השיניים ⚙️ ליד **Project Overview** (למעלה בתפריט השמאלי) → **Project settings**.
2. גלול למטה לאזור **Your apps**. כרגע הוא ריק.
3. לחץ על האייקון **`</>`** (Web).
4. **App nickname**: הקלד `web` → **אל** תסמן את "Also set up Firebase Hosting" → לחץ **Register app**.
5. יופיע לך קטע קוד. בתוכו יש בלוק שנראה כך:

```
const firebaseConfig = {
  apiKey: "AIzaSyB...ארוך...",
  authDomain: "residency-tracker-xxxxx.firebaseapp.com",
  projectId: "residency-tracker-xxxxx",
  storageBucket: "residency-tracker-xxxxx.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abcdef123456"
};
```

6. **העתק את הבלוק הזה** (מ-`{` ועד `}` כולל) — נדביק אותו בשלב הבא. אפשר להדביק בינתיים בפנקס רשימות.
7. לחץ **Continue to console**.

> 💡 ה-apiKey הזה **מיועד** להיות גלוי באתר — הוא לא סיסמה. האבטחה נעשית ע"י החוקים שפרסמת בשלב 4, ולכן אין בעיה שהוא יופיע בקוד ציבורי.

## חלק ב׳ — עריכת הקובץ

### שלב 6: הדבקת הקונפיג בתוך index.html

1. פתח את `index.html` בעורך טקסט:
   - **Windows**: קליק ימני על הקובץ → Open with → **Notepad** (או VS Code אם יש).
   - **Mac**: קליק ימני → Open With → **TextEdit** (ואז Format → Make Plain Text).
2. בתחילת הקובץ (בערך שורה 20) תמצא בלוק מסומן ב-‼️ שנראה כך:

```
const firebaseConfig = {
  apiKey: "PASTE_HERE",
  authDomain: "PASTE_HERE.firebaseapp.com",
  ...
};
```

3. **מחק את כל הבלוק הזה** — מהשורה `const firebaseConfig = {` ועד השורה `};` (כולל) — **והדבק במקומו** את הבלוק שהעתקת בשלב 5. שים לב שנשארת השורה `const firebaseConfig =` בתחילתו — אם העתקת רק את `{...}`, ודא שהשורה נראית בסוף כך:
   `const firebaseConfig = { apiKey: "AIza..., ... };`
4. **שמור** (Ctrl+S). ודא שהקובץ עדיין נקרא `index.html` ולא `index.html.txt` (ב-Notepad: Save As → File type: **All files**).

**בדיקה עצמית:** פתח את הקובץ בדפדפן (דאבל-קליק). אם רואים את מסך הכניסה עם כפתור Google — הקונפיג הודבק נכון. (הכפתור עצמו עוד לא יעבוד מקומית — זה תקין, נטפל בזה בשלב 10.) אם רואים "⚙️ נשאר רק לחבר Firebase" — ההדבקה לא נקלטה, חזור לשלב 6.

## חלק ג׳ — העלאה ל-GitHub

### שלב 7: חשבון GitHub (דלג אם יש לך)

1. גש ל-**https://github.com/signup**
2. הזן אימייל, סיסמה, ושם משתמש. **שם המשתמש יופיע בכתובת האתר** — בחר משהו סביר (למשל `raphael-md`).
3. אשר את האימייל.

### שלב 8: יצירת מאגר (Repository)

1. התחבר ל-GitHub → לחץ על ה-**+** בפינה הימנית העליונה → **New repository**.
2. **Repository name**: `residency-tracker` (באותיות קטנות, בלי רווחים).
3. בחר **Public** (חובה — GitHub Pages בחינם עובד רק על Public).
4. אל תסמן שום צ'קבוקס נוסף → לחץ **Create repository**.

### שלב 9: העלאת הקובץ

1. בדף הריק של המאגר, לחץ על הקישור **uploading an existing file** (בפסקה "Quick setup").
2. גרור את `index.html` (הגרסה **אחרי** שלב 6, עם הקונפיג!) לתוך אזור הגרירה. אפשר לגרור גם את `firestore.rules` ואת המדריך הזה — לא מזיק.
3. למטה, לחץ על הכפתור הירוק **Commit changes**.
4. ודא שהקובץ `index.html` מופיע ברשימת הקבצים של המאגר.

### שלב 10: הפעלת GitHub Pages (זה מה שנותן URL)

1. בדף המאגר, לחץ על הלשונית **Settings** (עם גלגל שיניים, בשורת הלשוניות העליונה).
2. בתפריט השמאלי, תחת "Code and automation", לחץ **Pages**.
3. תחת **Build and deployment**:
   - **Source**: השאר `Deploy from a branch`.
   - **Branch**: פתח את הרשימה שכתוב בה `None` → בחר **main** → לידו השאר **/(root)** → לחץ **Save**.
4. המתן 1–3 דקות ורענן את הדף. למעלה יופיע ריבוע:
   **"Your site is live at `https://USERNAME.github.io/residency-tracker/`"**
5. **העתק את הכתובת הזו** — זה ה-URL שלך. פתח אותה וודא שמסך הכניסה נטען.

### שלב 11: אישור הדומיין ב-Firebase — **חובה, בלי זה Google ייכשל**

Google מרשה התחברות רק מדומיינים שאישרת מראש:

1. חזור ל-**console.firebase.google.com** → הפרויקט שלך.
2. **Build → Authentication** → לשונית **Settings** (למעלה) → בתפריט הצדדי **Authorized domains**.
3. לחץ **Add domain**.
4. הקלד **בדיוק**: `USERNAME.github.io` (החלף USERNAME בשם המשתמש שלך; **בלי** `https://` ו**בלי** `/residency-tracker`).
5. לחץ **Add**.

## שלב 12: בדיקה סופית 🎉

1. פתח את הכתובת `https://USERNAME.github.io/residency-tracker/`.
2. לחץ **כניסה עם Google** → בחר חשבון → אשר.
3. מלא פרופיל: שם, התמחות (כירורגיה אורתופדית 😉), תאריכי התחלה וסיום, סטטוס שלב א׳ וב׳ → **שמירה והצגה**.
4. אמורים להופיע: טבעת האחוזים, ציר ההתמחות, הספירות לאחור.
5. **בדיקת השיתוף:** שלח את הקישור לחבר (או פתח בטלפון עם חשבון Google אחר). אחרי שהוא ימלא פרופיל — שניכם תופיעו בלוח החברים, אחד אצל השני, בלי לרענן.
6. כתבו הערה אחד לשני — היא תופיע מיד.

## עדכון האתר בעתיד

אם תרצה לשנות משהו בקוד:
1. בדף המאגר ב-GitHub לחץ על `index.html` → אייקון העיפרון ✏️ (Edit) → ערוך ישירות בדפדפן → **Commit changes**.
2. האתר מתעדכן אוטומטית תוך 1–2 דקות.

(או: מחק את הקובץ והעלה גרסה חדשה דרך Upload files.)
